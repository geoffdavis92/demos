01 What Is React?

From the React website…

A Javascript library for building user interfaces.

React.js is a streamlined and powerful library that can quickly create complex, interactive UIs, using pure Javascript.

It was created by engineers at Facebook to create resuable components that can integrate easily with user data. It was open-sourced in 2013, at which point it started gaining traction amongst the development community, and eventually becoming the (citation needed =>) most-used Javascript UI library in 2015.

React is...

...Component-Based

Everything building block in React is a component. Components have properties, can have methods written in to them, can have state, are reusable, and can hold other components.

Components have a lifecycle that determines things like when they render or if they get updated, and allows developers a huge amount of control over their applications' behavior.

...Declarative

Each component's properties and methods are written in their class. Using JSX syntax, you can compose components using a HTML-style Javascript code, where you can see the event handlers and inherited properties.

This allows developers to have an easy-to-read preview of what their rendered component will look like.

...Unidirectional

React's data-flow is unidirectional; any given component- by default- can inherit properties (props) and methods from their parent component. (but 2-way data binding is still possible)

This allows data to be easily traced and controlled as it moves to lower-level children components.

...Stack-Agnostic

React can be used by any tech stack, and can easily integrate with other libraries, tools, and frameworks.

Using Express.js or another node.js-based server framework, React can even be rendered server-side to increase application speed and enhance user experience.
